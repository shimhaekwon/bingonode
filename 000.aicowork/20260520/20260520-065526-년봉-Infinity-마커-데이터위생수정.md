# 세션 이력: 2026-05-20 — yfinance 부분 기간 row 데이터 위생 수정

## 작업 대상

- 페이지: `public/stock/node-index.html` 년봉(Y) 차트 표시
- 적용 원칙: 6-6 원칙

## 증상

SK Hynix(000660.KS) 년봉 차트에 다음 이상:
- 우측에 **"최저 0 (+Infinity%)"** 마커 표시
- Y축이 **-800,000** 까지 늘어나 캔들들이 상단으로 몰림
- 2026년 yearly 캔들이 비현실적으로 길게 늘어짐 (low=0 ~ high=1,995,000)

## 1. 원인 분석

### 직접 증거 — yfinance 응답 점검

`_fetchYahooCandles('000660.KS', { yahoo:'1mo', yearsBack:40 })` 실측:

```json
{"date":"2026-05-19","open":0,"high":0,"low":0,"close":1745000,"volume":0}
```

→ yfinance는 **진행 중 월/주봉**의 부분 row를 close만 채우고 OHL은 0으로 반환.

### 코드 경로

1. `_fetchYahooCandles` 의 기존 필터 `row.close !== null && row.date` — close가 1745000 (non-null)이라 통과.
2. `_aggregateToYearly` 의 2026 버킷이 이 row를 포함 → `Math.min(...months.map(m => m.low))` 에 0이 포함 → **`year.low = 0`**.
3. 프론트 `_getMarkers` 가 `(lastClose - 0) / 0 * 100 = Infinity` 계산.
4. LightweightCharts auto-scale 이 0~1,995,000 캔들을 그리며 Y축을 음수까지 확장.

년봉 집계 결과 확인:
```json
{"date":"2026-01-01","open":870000,"high":1995000,"low":0,"close":1745000,"volume":332840010}
```
low=0 재현 확인.

## 2. 수정 설계

| 위치 | 변경 |
|---|---|
| `libs/stockFetcher.js` `normalizeOhlcv()` (신규 모듈 함수) | close>0인데 OHL 중 하나라도 ≤0 이면 **4개 모두 close로 정규화**(degenerate doji). 진행 중 기간의 "현재 종가"는 보존, 다운스트림 0 division 방지 |
| `_fetchYahooCandles` (W/M/Y 경로) | `.map(normalizeOhlcv).filter(close>0 && date)` |
| `fetchFromYahoo` (daily 경로) | 동일 정규화 적용 — 방어 일관성 |
| `public/stock/js/stock-chart.js` `_getMarkers` | **표시 방어**: `high<=0` 시 최고 마커 생략, `low<=0` 시 최저 마커 생략. 데이터 위생 통과 후에도 만일 대비 |

### 설계 근거

- **소스 정규화** 가 가장 깨끗: 진입 시점에서 처리하면 집계·MA·마커·차트가 자동 정상화.
- **close 보존**: 진행 중 기간의 "오늘 종가"는 사용자에게 유용한 정보 → 제거하지 않고 OHLC 모두 close로 collapse → doji로 표시.
- **표시 방어** 는 defense-in-depth: 향후 정규화 누락 시에도 마커가 깨지지 않음.

## 3. 수정 상세

### 3.1 `libs/stockFetcher.js` — `normalizeOhlcv` 함수 신설

```js
function normalizeOhlcv(row) {
  if (!row || !row.date) return null;
  const c = row.close;
  if (c == null || !(c > 0)) {
    return { date: row.date.toISOString().split('T')[0],
             open: row.open, high: row.high, low: row.low, close: row.close,
             volume: row.volume };
  }
  let o = row.open, h = row.high, l = row.low;
  if (!(o > 0) || !(h > 0) || !(l > 0)) { o = c; h = c; l = c; }
  return { date: row.date.toISOString().split('T')[0],
           open: o, high: h, low: l, close: c, volume: row.volume || 0 };
}
```

호출처 변경:
```diff
- .filter(row => row.close !== null && row.date)
- .map(row => ({date:..., open:row.open, ...}));
+ .map(row => normalizeOhlcv(row))
+ .filter(row => row && row.close > 0 && row.date);
```
`fetchFromYahoo`(daily) 도 동일 패턴 적용.

### 3.2 `public/stock/js/stock-chart.js` — `_getMarkers` 방어

```diff
- const highPct = ((lastClose - data[highIdx].high) / data[highIdx].high * 100).toFixed(2);
- const lowPct  = ((lastClose - data[lowIdx].low)  / data[lowIdx].low  * 100).toFixed(2);
- const markers = [{...최고}, {...최저}];
+ const markers = [];
+ const highVal = data[highIdx] && data[highIdx].high;
+ if (highVal > 0) { markers.push({...최고 with sign}); }
+ const lowVal  = data[lowIdx]  && data[lowIdx].low;
+ if (lowVal > 0)  { markers.push({...최저 with sign}); }
```
