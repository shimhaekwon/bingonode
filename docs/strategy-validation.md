# 전략 검증 보고서 — Mandel 인기도 패널티 도입 근거

## TL;DR

- **빈도 분석(Hot/Cold)은 적중률 면에서 무작위와 통계적으로 동등하다.** 1222회 한국 로또 실측 검증.
- **Mandel "비인기 번호" 가설은 데이터로 입증된다** — 인기 번호 조합이 1등에 당첨될 때 동점자 수가 23.9% 더 많다.
- 따라서 `bingoscript.js`의 `scoreNumbers()`에 인기도 패널티 가중치(`HP.gamma`)를 추가했다.

## 검증 내역

검증 코드 및 결과: 자매 프로젝트 [`bingo` 레포](../../bingo/compare_strategies.py).

### 1. Mandel 가설 — 인기도 vs 1등 동점자 수

전체 1222회 1등 당첨 데이터를 인기도 4분위로 분할.

| 분위 | 평균 인기도 | 평균 1등 당첨자 수 | 표본 |
|---|---|---|---|
| Q1 (비인기) | 1.274 | **7.6명** | 302 |
| Q2 | 1.376 | 9.0명 | 302 |
| Q3 | 1.446 | 8.3명 | 302 |
| Q4 (인기) | 1.551 | **9.4명** | 302 |

**해석**: 단조 증가는 아니지만 Q1↔Q4에서 +23.9% 격차. 1등이 났을 때 비인기 번호 조합일수록 1인당 수령액이 약 1/1.24배 더 크다는 EV 우위가 존재.

### 2. 전략 백테스트 (500회 표본)

| 전략 | ROI | 평균 인기도 | 1등 | 2등 | 3등 | 4등 | 5등 |
|---|---|---|---|---|---|---|---|
| hot_cold (빈도) | 11.7% | 1.421 | 0 | 0 | 0 | 3 | 52 |
| **mandel** (비인기 우선) | **13.4%** | **1.000** | 0 | 0 | 0 | 3 | **64** |
| hybrid (혼합) | 9.7% | 1.229 | 0 | 0 | 0 | 0 | 68 |

**관찰**:
- 적중률은 통계적으로 동등 — 독립 시행이므로 예측대로
- **mandel이 hot_cold 대비 5등 적중률 +23%** (52→64장) — 빈도 분석이 노이즈를 따라간 가능성 시사
- **hybrid가 가장 나쁨** — 두 휴리스틱을 섞으면 4등 0장으로 악화. 신호 충돌

## 코드 적용

### `services/mandelPopularity.js` (신규)
공용 인기도 계산 모듈.
- `popularity(n)`: 단일 번호 인기도
- `popularityVector()`: 1~45 인기도 벡터
- `popularityZ()`: 정규화된 인기도 (z-score)
- `combinationPopularity(nums)`: 6수 조합 평균 인기도

가중치:
- `n ≤ 31`: ×1.5 (생일·기념일 편향)
- `n ∈ {7,14,21,28,35,42}`: ×1.3 (7배수 행운수)
- `n ∈ {3,9,13}`: ×1.15 (추가 행운수)

### `public/bingo/bingoscript.js` (수정)
- `HP.gamma = 0.4` 신설 — Mandel 인기도 패널티 가중치
- `scoreNumbers()` 마지막에 `S[n] -= HP.gamma * popZ[n]` 차감

### 운영 가이드 — `HP.gamma` 튜닝

| 값 | 효과 |
|---|---|
| 0 | 비활성 (기존 빈도 기반과 동일) |
| 0.2~0.4 | **권장**. 강한 빈도 신호는 살리고 약한 동점에서 비인기 우선 (타이브레이커) |
| 0.6~1.0 | Mandel 우위. 빈도 신호 약화. 백테스트에서 ROI 약간 우위 보였던 영역 |
| > 1.0 | 빈도 무시, 사실상 항상 32~45 위주 추천 |

`hybrid` 백테스트가 가장 나빴다는 점을 고려하면 **두 신호를 동등하게 섞기보다는 한쪽이 명확히 주도**하도록 두는 것이 안전. 기본값 0.4는 빈도가 주신호로 남되 인기도가 타이브레이커 역할.

## 한계

- **적중 확률은 불변**. 어떤 가중치 조합도 813만분의 1을 바꾸지 못함.
- Mandel의 EV 효과(+23.9%)는 **1~3등에 적중했을 때만** 발현. 5등(3개 일치) 50,000원/5,000원은 고정.
- 본 백테스트는 500회 — 1~3등 적중 0회였으므로 **실측 ROI 차이는 표본 변동에 가까움**. 다만 이론적으로 EV 우위는 있고, 적용 비용은 0(코드 수정만)이므로 채택할 가치 있음.

## 미적용 / 보류 항목

- `scoreNumbers`의 빈도 윈도우(`W`), 갭 분석(`computeTypeGapAnalysis`), 전략 선택(`selectOptimalStrategy`)은 **현재 그대로 유지**. 이들은 적중률에 영향 없는 노이즈 성격이지만, 사용자 친화적 분석 패널 데이터로 가치가 있어 제거하지 않음.
- 백엔드 `bingoController.js`의 `getPredict` 엔드포인트는 별도의 미해결 결함 존재 (`getRecent`, `getOne`, `generatePredictions` 미정의 ReferenceError). 본 개선과 별개 이슈로 분리.
